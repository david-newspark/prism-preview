What's in this build
📋
Issue Queue
Browse open GitHub (or Linear) issues. One click to start a session on any issue.
⌨️
AI Collab Terminal
Full xterm.js terminal running Claude Code. Multiple concurrent sessions, each isolated and persisted.
✅
Plan Approval Gate
Claude surfaces its plan inline before touching any code. You approve, reject, or comment per step.
🔀
PR Panel
When a PR opens, review the diff and merge without leaving the app. Ship-all mode for batch merges.
📊
Agent Progress Tree
Live structured view of what Claude is doing — skill runs, sub-steps, and status as they happen.
📈
Insights
Session analytics grouped by issue — token usage, stage timings, and project-level stat cards.
🕒
Session History
Every session persisted locally. Search, pin, archive, and restore. Survives restarts and updates.
⚙️
Settings
Custom data dir, model tiers, auto-approve level, PR rules scoped to project or global.
🎨
Two Themes
Forge (warm amber) and Obsidian (cool violet). Switchable live from the toolbar.
🏠
Fully Local
Native binary + embedded WebView. No Electron, no cloud infra, no Anthropic API key beyond Claude Code itself.
🔍
Call Graph Panel
PR review now shows an interactive call graph (Go + JS/TS). Select a node and hit "Explain with AI" to send a targeted prompt to the active session.
✏️
Inline Code Editor
Edit files directly inside the PR review panel for surgical fixes post-review — no context switch needed before merging.
⚡
RTK Integration
Settings → AI Tools shows RTK (token optimizer) status. Insights shows a live savings chart — how many tokens RTK is cutting per command type.
Prerequisites
Claude Code CLI — installed and authenticated (claude on your PATH). Get it at claude.ai/code.
Git configured with access to your GitHub org (needed for PR creation and issue listing).
GitHub CLI (gh) authenticated — PRISM uses it for issue and PR operations.
A GitHub project to point PRISM at. You'll set this in Settings on first launch — just enter your org/repo.
Default config

PRISM creates ~/.prism/config.yaml on first run. Zero config required to get started — but here's what you can tune:

# ~/.prism/config.yaml

provider: "github" # or "linear"

skip_permissions: true # passes --dangerously-skip-permissions

model_tiers:
  light: claude-haiku-4-5-20251001
  standard: claude-sonnet-4-6
  deep: claude-sonnet-4-6 # swap for opus on complex work

automation:
  auto_approve_level: 0 # 0 = all gates need human approval
